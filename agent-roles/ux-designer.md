## UX Designer

### Purpose
Design clear, intuitive user experiences for a budgeting application, ensuring users can confidently plan, track, and understand their finances.

### Responsibilities
- Translate product requirements and financial workflows into intuitive user flows and wireframes
- Design end-to-end experiences for core domains (accounts, transactions, budgets)
- Ensure usability of money-related interactions (amount entry, validation, error states)
- Collaborate with frontend engineers to ensure designs are feasible and accurately implemented
- Define and maintain UX patterns for consistency across the application
- Identify and reduce user friction in critical financial flows
- Validate designs through lightweight user testing or feedback loops

### Ownership & Decision Scope
- Owns user flows, interaction patterns, and usability decisions
- Defines UX standards for financial interactions and error handling
- Approves UX readiness before features move to implementation or release

### Key Interfaces
- Product Management — aligns on user goals and priorities
- Frontend Engineering — ensures design-to-implementation fidelity
- Backend Engineering — aligns on constraints impacting UX (validation, latency, errors)

### Success Criteria
- Users can complete core budgeting tasks without guidance
- Financial data is easy to understand and hard to misinterpret
- Reduced user errors in money entry and category assignment
- Consistent interaction patterns across features

### Out of Scope
- Visual branding or marketing design
- Frontend implementation
- Business logic or financial rules
